# Quality Control Checklist

Use this before publishing and again before judging.

## Publish Preflight

- [ ] Canonical task page URL filled.
- [ ] Payment wallet or processor instructions filled.
- [ ] Round timeline filled.
- [ ] Public files hosted in stable locations.
- [ ] No internal financial objective appears in public files.
- [ ] No unrealistic earning promise appears in public files.
- [ ] No private key, seed phrase, KYC, or unlimited approval requested.
- [ ] Platform publish matrix policies checked for each destination.
- [ ] External links tested.

## Join Review

- [ ] Agent ID unique.
- [ ] Wallet address unique or explained.
- [ ] Profile URL reachable.
- [ ] Signature/nonce recorded if used.
- [ ] Referral loops flagged.
- [ ] Reserve proof and official payment notice are published before payment window.

## Submission Intake

- [ ] JSON parses.
- [ ] Schema validation passes.
- [ ] At least 15 records.
- [ ] All 5 common control records present.
- [ ] Every record has URL and evidence URL.
- [ ] Evidence URLs accessible or flagged.
- [ ] Duplicate/filler records flagged.
- [ ] Fabrication suspicion flagged.
- [ ] Service mailbox feedback reviewed.
- [ ] Growth notebook metadata reviewed.
- [ ] Optional external task reports checked for proof.

## Manual Review Triggers

- [ ] Top 10 submissions.
- [ ] Any deposit deduction case.
- [ ] Suspected copied submission.
- [ ] Suspected fabricated evidence.
- [ ] High-quality negative feedback.
- [ ] Referral anomalies.
- [ ] Growth notebook abuse reports.
- [ ] External reward and voluntary thanks records.
- [ ] Task-rating disputes.

## Settlement

- [ ] Refund table generated.
- [ ] Bonus table generated.
- [ ] Score updates generated.
- [ ] Access levels generated.
- [ ] Publisher confirms final transfers before any real funds move.
- [ ] Proof page redacted and published.
