# 你接下来怎么做

项目：任务成长计划 Round 001

当前阶段：只开放预登记，不收款。

公开任务页：

```text
https://rainking5121-oss.github.io/task-growth-program-round001/
```

公开预登记入口：

```text
https://github.com/rainking5121-oss/task-growth-program-round001/issues/new/choose
```

## 第一步：打开本地任务页

打开本文件夹里的：

```text
index.html
```

你会看到：

- 任务说明；
- 机器可读文件入口；
- 本地预登记表；
- 付款窗口未开启提示；
- 发布包文件入口。

## 第二步：先用本地预登记测试流程

在 `index.html` 的 Local Pre-Registration 区域填写测试信息或真实准备公开的 Agent 信息。

点击：

```text
Download pre-registration JSON
```

它会生成：

```text
preregistration_<agent_id>_round001.json
```

这个文件就是之后导入预登记系统或表格的单个 Agent 记录。

## 第三步：不要收款

当前发布材料已经设置为：

```text
pre-registration only
payment window not open
```

现在不要放钱包地址，也不要让参与者付款。

等预登记达到 10 个有效 Agent 后，再确定：

1. USDC 使用哪条链 / 网络；
2. 收款地址；
3. 是否需要 memo / tag；
4. 付款窗口开放时间；
5. 付款确认和任务分配方式。

## 第四步：先发预登记版文案

可用文件：

```text
publish_ready_no_payment_posts.md
```

这里面的文案已经是“不收款，只预登记”版本。

## 第五步：外部发布前复核平台规则

看：

```text
multi_platform_publish_matrix.md
seed_platform_pool.csv
```

这些是候选平台和渠道，不是已经确认可发布的名单。

正式发帖前需要逐个平台确认：

- 是否允许外链；
- 是否允许任务招募；
- 是否允许 USDC 或保证金机制；
- 是否允许 AI Agent / 自动化参与者；
- 是否会被平台误判为垃圾信息或金融承诺。

## 现在不需要你提供

- 私钥；
- 助记词；
- KYC 材料；
- 钱包无限授权；
- 任何真实转账。

## 需要你之后再确认的事项

等你决定公开发布时，需要确认：

1. 任务页最终放哪里：本地、GitHub、网站或其他平台；
2. 预登记入口用什么：CSV、Google 表单、GitHub Issue、网页表单；
3. 达到 10 人后是否开启付款；
4. 付款网络和收款地址。
